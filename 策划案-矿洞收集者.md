# 外部信息

## 目标

### 创造体验
**核心体验**：在有限资源（行动点）约束下的风险决策与资源管理
- **短期决策**：每次挖掘都需要权衡"消耗 vs 收益"
- **中期目标**：在行动点耗尽前找到陷阱门进入下层，或找到升降梯安全撤退
- **长期成长**：通过积累资源进行永久升级，降低后续挖掘难度

**情感曲线**：
1. **探索期待**：每层洞穴随机生成，未知区域充满可能性
2. **策略紧张**：行动点逐渐减少，需要计算最优路径
3. **风险刺激**：选择"继续深入"还是"安全撤退"的博弈
4. **成长满足**：升级后能力提升，挑战更深层次

## 商业化

### 目标人群
- **Game Jam 参与者/观众**：对 Ludum Dare 48 小时创作的作品感兴趣
- **Roguelike 爱好者**：喜欢随机生成、永久死亡、渐进式成长的玩家
- **轻度策略玩家**：喜欢资源管理和风险决策，但不需要复杂操作

### 场景
- **PC 单机游戏**：鼠标点击操作，碎片时间游玩（单次 3-10 分钟）
- **Web 平台**：itch.io 发布，支持浏览器直接游玩
- **免费游玩**：无商业化计划，纯粹的 Game Jam 作品

## 竞品/参考
- **Minesweeper（扫雷）**：网格揭示机制、数字提示系统
- **FTL: Faster Than Light**：资源管理 + 风险决策 + Roguelike 循环
- **Slay the Spire**：永久升级系统 + 单局随机性
- **挖矿类 Roguelike**：《地牢挖掘者》《矿洞探险》等

# 游戏结构

## 功能概览

### 功能框架（有什么的构成）

```
矿洞收集者
├── 核心玩法系统
│   ├── 挖矿系统（行动点消耗、矿物挖掘）
│   ├── 资源收集系统（铁、金、钻石、血矿）
│   ├── 关卡深入系统（陷阱门、升降梯）
│   └── 揭示机制（3x3 已挖掘区域 + 12 格可挖掘边界）
├── 关卡生成系统
│   ├── 随机洞穴生成（权重分布）
│   ├── 保证路径可达（入口 → 陷阱门）
│   └── 深度递进（不同层级的矿物分布）
├── 成长系统
│   ├── 镐子升级（降低挖掘消耗）
│   ├── 治疗升级（进入下层时额外恢复行动点）
│   └── 生命值升级（提高初始最大行动点）
├── UI 系统
│   ├── 游戏内 HUD（行动点、层数、资源计数）
│   ├── 升级商店界面（显示升级选项和消耗）
│   └── 提示系统（数字标签显示矿物价值）
└── 音效系统
    ├── 挖掘音效（不同矿物）
    ├── 进入下层音效
    ├── 失败音效
    └── 升级音效
```

### 系统循环（做什么的流程）

**核心循环（单局游戏）：**
```
开始新局
    ↓
[第1层] 点击入口开始 → 揭示周围区域
    ↓
消耗行动点挖掘矿物 → 收集资源（铁/金/钻石/血矿）
    ↓
决策分支：
    ├─→ 找到陷阱门 → 进入下一层（恢复部分行动点）→ 重复挖掘循环
    ├─→ 找到升降梯 → 退出矿洞 → 保存所有资源 → 进入商店
    └─→ 行动点耗尽 → 游戏失败 → 失去所有资源 → 返回商店
```

**元游戏循环（多局成长）：**
```
[商店] 使用积累的资源购买升级
    ↓
永久能力提升（镐子/治疗/生命值）
    ↓
开始新局（带着升级的能力）
    ↓
更容易深入更深层，收集更多资源
    ↓
返回商店进行更多升级
    ↓
（循环往复，逐步解锁所有升级）
```

## 功能详情

### 1. 挖矿系统

#### 1.1 行动点机制
**设计意图**：核心约束资源，所有玩家决策围绕行动点展开

**数值配置：**
- **初始最大行动点**：7 点（可通过升级提升）
- **进入下层恢复**：+2 点（可通过升级提升）
- **血矿恢复**：+5 点

**消耗规则：**
| 矿物类型 | 基础消耗 | 可升级 | 说明 |
|---------|---------|--------|------|
| 石头 | 1 点 | ✅ | 最常见，升级后消耗 0 点 |
| 铁矿 | 2 点 | ❌ | 升级货币 |
| 金矿 | 7 点 | ✅ | 高价值，升级后消耗 6 点 |
| 钻石 | 9 点 | ✅ | 最高价值，升级后消耗 8 点 |
| 血矿 | 0 点 | ❌ | 回复 5 点行动力 |
| 陷阱门 | 0 点 | ❌ | 进入下层 |
| 升降梯 | 0 点 | ❌ | 退出矿洞 |

**失败条件：**
- 行动点不足以挖掘任何相邻的矿物 → 游戏失败 → 失去所有本局收集的资源

#### 1.2 挖掘交互
**操作方式**：鼠标左键点击可挖掘的单元格

**可挖掘判定：**
1. 单元格必须被 `coverer` 层遮罩（未被挖掘）
2. 单元格必须在已揭示区域的边缘（相邻已挖掘区域）
3. 玩家行动点 ≥ 该单元格的挖掘消耗

**挖掘结果：**
1. 消耗对应行动点
2. 移除 `coverer` 遮罩，显示矿物
3. 移除 `hider` 遮罩，显示周围提示数字
4. 若挖到资源矿物（铁/金/钻石/血矿），增加对应资源计数
5. 更新周围单元格的提示数字
6. 播放对应音效

#### 1.3 揭示机制
**初始揭示（第一次点击）：**
- **3x3 区域**：完全揭示（相当于已挖掘），显示矿物和提示数字
  - 移除 `coverer` 层（显示矿物）
  - 移除 `hider` 层（显示提示数字）
- **外围 12 格**：仅移除 `coverer` 层，成为可挖掘区域
  - 不包含四个斜角（需要挖掘相邻格子才能逐步揭示）

**后续揭示**：
- 每次挖掘后，新挖掘的格子周围的格子自动更新提示数字

**提示数字规则**：
- 显示该单元格矿物的"价值"（与消耗相同）
- 用于帮助玩家判断挖掘成本

### 2. 资源收集系统

#### 2.1 资源类型
| 资源名称 | 英文名 | 获取方式 | 主要用途 |
|---------|--------|---------|---------|
| 铁矿 | Iron | 挖掘白色矿石 | 升级镐子、生命值 |
| 金矿 | Gold | 挖掘黄色矿石 | 升级镐子 |
| 钻石 | Diamond | 挖掘蓝色矿石 | 升级镐子、治疗 |
| 血矿 | Blood | 挖掘红色矿石 | 升级治疗、生命值 |

#### 2.2 资源存储规则
**单局临时存储**：
- 在矿洞内收集的资源存储在 `gained_minerals` 字典
- 仅在成功退出（通过升降梯）时才保存到永久资源

**永久资源存储**：
- 通过 `ResourceManager` 管理
- 在商店界面可见当前拥有的资源
- 用于购买升级

**失败惩罚**：
- 行动点耗尽导致失败 → `gained_minerals` 清空 → 所有本局资源丢失

### 3. 关卡生成系统

#### 3.1 洞穴生成参数
**网格尺寸**：12 列 x 10 行 = 120 个单元格

**保证元素**：
- **升降梯（Lifter）**：玩家点击的入口位置
- **陷阱门（Trapdoor）**：随机生成，与升降梯距离 ≥ 5 格
- **保证路径**：升降梯 → 陷阱门之间必有可通行路径

#### 3.2 矿物分布权重
**随深度变化**：越深层，高价值矿物越多

**第 1 层权重**：
| 矿物 | 权重 | 占比 |
|-----|------|------|
| 石头 | 27 | 56.3% |
| 铁矿 | 11 | 22.9% |
| 金矿 | 1 | 2.1% |
| 钻石 | 0 | 0% |
| 血矿 | 9 | 18.8% |

**第 2 层权重**：
| 矿物 | 权重 | 占比 |
|-----|------|------|
| 石头 | 20 | 48.8% |
| 铁矿 | 12 | 29.3% |
| 金矿 | 4 | 9.8% |
| 钻石 | 0 | 0% |
| 血矿 | 5 | 12.2% |

**第 3 层权重**：
| 矿物 | 权重 | 占比 |
|-----|------|------|
| 石头 | 20 | 40.8% |
| 铁矿 | 10 | 20.4% |
| 金矿 | 14 | 28.6% |
| 钻石 | 1 | 2.0% |
| 血矿 | 4 | 8.2% |

**第 4 层权重**：
| 矿物 | 权重 | 占比 |
|-----|------|------|
| 石头 | 16 | 42.1% |
| 铁矿 | 2 | 5.3% |
| 金矿 | 16 | 42.1% |
| 钻石 | 4 | 10.5% |
| 血矿 | 0 | 0% |

**第 5+ 层权重**：
| 矿物 | 权重 | 占比 |
|-----|------|------|
| 石头 | 10 | 32.3% |
| 铁矿 | 0 | 0% |
| 金矿 | 10 | 32.3% |
| 钻石 | 11 | 35.5% |
| 血矿 | 0 | 0% |

**设计意图**：
- 前期（1-2 层）：大量石头和铁矿，少量金矿，较多血矿（帮助新手存活）
- 中期（3-4 层）：金矿增加，开始出现钻石，血矿减少（难度上升）
- 后期（5+ 层）：高价值矿物为主，无血矿（高风险高回报）

#### 3.3 生成算法
1. **创建空网格**（12x10）
2. **放置升降梯**（玩家点击位置）
3. **放置陷阱门**（随机位置，距离升降梯 ≥ 5 格）
4. **生成保证路径**（A* 或随机游走，确保可达）
5. **填充其余单元格**（根据当前层级权重随机生成）

### 4. 升级系统

#### 4.1 升级类型

**镐子升级（Axe Upgrade）**：降低挖掘消耗
| 等级 | 效果 | 消耗 |
|-----|------|------|
| 1 | 挖掘石头消耗 -1（0 点） | 铁 x5 |
| 2 | 挖掘金矿消耗 -1（6 点） | 铁 x5, 金 x10 |
| 3 | 挖掘钻石消耗 -1（8 点） | 铁 x10, 金 x20, 钻石 x20 |

**治疗升级（Heal Upgrade）**：进入下层时额外恢复行动点
| 等级 | 效果 | 消耗 |
|-----|------|------|
| 1 | 下层恢复 +1（总共 +3） | 血 x10 |
| 2 | 下层恢复 +1（总共 +4） | 血 x30, 钻石 x5 |
| 3 | 下层恢复 +1（总共 +5） | 血 x60, 钻石 x10 |
| 4 | 下层恢复 +1（总共 +6） | 血 x80, 钻石 x15 |
| 5 | 下层恢复 +1（总共 +7） | 血 x80, 钻石 x20 |

**生命值升级（Health Upgrade）**：提高初始最大行动点
| 等级 | 效果 | 消耗 |
|-----|------|------|
| 1 | 最大行动点 +1（总共 8 点） | 铁 x5, 血 x5 |
| 2 | 最大行动点 +1（总共 9 点） | 铁 x20, 金 x5, 血 x5 |
| 3 | 最大行动点 +1（总共 10 点） | 铁 x40, 金 x12, 血 x5 |
| 4 | 最大行动点 +1（总共 11 点） | 铁 x60, 金 x20, 血 x5 |

#### 4.2 升级购买流程
1. 玩家从矿洞成功退出（通过升降梯）
2. 进入商店界面，显示当前资源和可购买的升级
3. 点击升级按钮 → 检查资源是否足够
4. 资源足够 → 扣除资源 → 应用升级效果 → 播放升级音效 → 刷新 UI
5. 资源不足 → 无反应或提示（待实现）
6. 升级到最大等级 → 隐藏对应升级选项

#### 4.3 升级优先级建议
**新手推荐路线**：
1. **镐子 Lv.1**（石头消耗 0 点）→ 性价比最高，前期探索更自由
2. **生命值 Lv.1-2**（总行动点 8-9）→ 提高容错率
3. **镐子 Lv.2**（金矿消耗 6 点）→ 提升收益效率
4. **治疗 Lv.1-2**（下层恢复 3-4 点）→ 支持更深层探索

### 5. UI 系统

#### 5.1 游戏内 HUD
**行动点显示**：
- 位置：屏幕左上角
- 格式：`当前值 / 最大值`（例如：7/10）
- 颜色：
  - 绿色：行动点 > 50%
  - 黄色：行动点 20-50%
  - 红色：行动点 < 20%

**层数显示**：
- 位置：屏幕左上角（行动点下方）
- 格式：`Layer: X`
- 作用：让玩家了解当前深度

**资源计数器**：
- 位置：屏幕右上角
- 显示：本局收集的铁/金/钻石/血矿数量
- 图标 + 数字形式

#### 5.2 商店界面
**布局**：水平排列三个升级选项（镐子、治疗、生命值）

**每个升级卡片显示**：
- 升级名称和图标
- 效果描述（英文）
- 消耗资源（图标 + 数字）
- 购买按钮
- 达到最大等级后隐藏

**当前资源显示**：
- 位置：商店界面顶部
- 显示玩家当前拥有的所有资源

#### 5.3 提示系统
**矿物价值数字**：
- 在矿物上方显示数字标签
- 数字 = 矿物的挖掘消耗（也代表价值）
- 通过 `labeler` TileMapLayer 实现

**特殊标记**：
- 陷阱门：显示 "100"（表示高价值/里程碑）
- 升降梯：显示 "0"（表示免费退出）
- 血矿：显示 "5"（表示恢复量）

## 游戏状态机

### 挖矿状态（MineState）
```
WAITING_START（等待开始）
    ↓ 玩家点击任意位置
PROCESSING（进行中）
    ↓ 行动点耗尽
FAIL（失败）
    ↓ 玩家点击重新开始
WAITING_START
```

### 游戏整体状态（GameState）
```
IDLE（闲置）
    ↓ 进入矿洞
PLAYING（游玩中）
    ↓ 成功退出 / 失败
GAME_OVER（结束）
    ↓ 返回商店
IDLE
```

## 数值需求

### 核心数值表

**矿物基础数值**：
| 矿物 | 消耗 | 价值 | 稀有度 |
|-----|------|------|--------|
| 石头 | 1 | 1 | 常见 |
| 铁矿 | 2 | 2 | 常见 |
| 金矿 | 7 | 7 | 稀有 |
| 钻石 | 9 | 9 | 极稀有 |
| 血矿 | 0 | 5（恢复） | 罕见 |

**行动点数值**：
| 参数 | 基础值 | 可升级范围 |
|-----|--------|-----------|
| 初始最大行动点 | 7 | 7 - 11 |
| 进入下层恢复 | 2 | 2 - 7 |
| 血矿恢复 | 5 | 固定 |

**升级成本曲线**：
- 镐子：前期便宜（铁 x5）→ 后期昂贵（铁 x10 + 金 x20 + 钻石 x20）
- 治疗：主要消耗血矿（10 → 30 → 60 → 80）+ 少量钻石
- 生命值：持续消耗铁（5 → 20 → 40 → 60）+ 少量金和血

### 数值平衡目标
1. **单局时长**：3-10 分钟（根据玩家策略和运气）
2. **首次成功退出**：预计 2-3 局后（新手学习曲线）
3. **全升级解锁**：预计 10-20 局（长期目标）
4. **单局平均收益**：
   - 成功退出（3-5 层）：铁 x10-20, 金 x5-10, 钻石 x1-3, 血 x5-10
   - 失败（1-2 层）：0（全部丢失）

## 技术架构

### 文件结构
```
ld-58-collector/
├── Framework/                    # 通用游戏框架（可迁移）
│   ├── Core/
│   │   ├── game_config.gd       # 游戏配置基类
│   │   ├── game_state.gd        # 游戏状态基类
│   │   ├── resource_manager.gd  # 资源管理器
│   │   └── event_system.gd      # 事件系统
│   ├── Grid/
│   │   ├── grid_game_base.gd    # 网格游戏基类
│   │   ├── mine_game_base.gd    # 挖矿游戏基类
│   │   ├── grid_cell.gd         # 单元格数据结构
│   │   ├── grid_pattern.gd      # 网格模式库
│   │   └── grid_layer_manager.gd # 图层管理器
│   ├── Generation/
│   │   ├── procedural_generator_base.gd # 程序生成基类
│   │   └── weighted_generator.gd # 权重生成器
│   └── Upgrade/
│       ├── upgrade_definition.gd # 升级定义资源
│       └── upgrade_manager.gd    # 升级管理器
├── Global/                       # 游戏全局系统
│   ├── game.gd                  # 游戏协调器（向后兼容层）
│   ├── game_constants.gd        # 游戏常量
│   ├── game_config.gd           # 游戏配置
│   ├── game_state.gd            # 游戏状态
│   ├── event_bus.gd             # 事件总线
│   └── sound_manager.gd         # 音效管理器
├── Resource/
│   └── cave_generator.gd        # 洞穴生成器
├── Scene/
│   ├── mine_game.tscn           # 主游戏场景
│   └── level_up.gd              # 商店界面
├── mine_game.gd                 # 主游戏逻辑
└── upgrate.gd                   # 升级卡片 UI
```

### 继承架构
```
Node（Godot 基类）
    ↓
GridGameBase（通用网格游戏）
    ↓
MineGameBase（挖矿游戏基类）
    ↓
mine_game.gd（矿洞收集者具体实现）
```
